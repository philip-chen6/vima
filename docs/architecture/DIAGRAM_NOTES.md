# Diagram Notes

## How Paper-Style Architecture Diagrams Are Usually Made

The diagram style in many ML papers is usually vector art, not generated by the
model itself. Common workflows:

1. **Figma / Illustrator / Keynote / PowerPoint**
   Draw rectangles, arrows, repeated blocks, and image crops manually. Export as
   SVG/PDF/PNG. This is fastest for a polished paper figure.

2. **TikZ / LaTeX**
   Program the figure with exact coordinates. This is best for papers where
   fonts, line widths, and reproducibility matter.

3. **diagrams.net / Excalidraw**
   Fast for architecture sketches. Less polished than Figma/TikZ, but easy to
   edit and version in the repo.

For VINNA, keep `.excalidraw` source files in `docs/architecture/` while the
architecture is still changing. When the system stabilizes, recreate the final
paper/demo figure in Figma or TikZ and export a static SVG/PDF.

## Figure Direction

The right visual metaphor is not a robot-control labeler. It is a **video
evidence encoder**:

```text
bodycam frames
  -> adaptive frame tokens
  -> depth / pose / VLM / tracking evidence reps
  -> repeated evidence-fusion block
  -> Scene Ledger + event timeline + benchmark heads
```

The benchmark head uses future frames and human-reviewed labels as ground truth.
